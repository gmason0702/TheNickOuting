# Design personalized RSVP link & token scheme

## Question

Each invitee gets a unique link (no login) that identifies them and lets them RSVP. How is that token generated, how long/opaque does it need to be, how does the page look it up against the Sheet row, and how do we stop someone from guessing another invitee's link or replaying/tampering with it? Decide the token format and the verification approach the RSVP page uses.
